Recipal

Skills used: AngularJS, Ionic, PhoneGap, NodeJS, MongoDB

Github Amazon App Store

Recipal is a mobile application allowing users to upload and view recipes in a step-wise manner on their phones. The idea came from how most recipes on the web are hard to follow and overwhelm the user with information. As a result, Recipal was created to guide beginners through a recipe without large amounts of information. The front-end is written in AngularJS and the back-end is written in NodeJS. Ionic is used for the UI and phonegap is used to integrate the application onto to the mobile device. The server (though not running currently) is ran on Amazon EC2. The client uses the MVC model and the server uses an event-driven architecture. HTTP requests and responses are used to communicate between the client and the server.

Bike Rack Application

Skills used: Java, GWT, Facebook API

Github

Used GWT to implement a web application allowing users to schedule and plan events for bike riding around Vancouver. The app uses google maps to locate available bike-racks across vancouver. Each event is based on these bike-rack locations around vancouver. User information is accessed through their google account. Users can also use facebook to post their events and tell their friends about interesting bike events across vancouver. The UI and back-end are both programmed using Java using the GWT framework, which transforms Java code into Javascript.

HTTP proxy

Skills used: C, HTTP Request/Response

Implemented a HTTP proxy using C. To do this, the application reads the HTTP request and response headers and then forwards the requests to the destination. Both content-length and chunk-encoding were deciphered. The proxy stores the requests and responses into a cache and depending on how recent a request is made, a cached response is sent back. The server also keeps track of a list of blacklisted IPs and sends a 400 code to any requests made from blacklisted IPs. Data was read through a buffer that the socket streams data into.
